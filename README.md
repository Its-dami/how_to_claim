# 我要怎么诉   

📖 项目简介 (Introduction)  
“我要怎么诉” 是一个专为法律人打造的实务级 AI 诉讼攻防教练（Skill/Prompt）。它摒弃了空泛的法律理论说教，严格植根于中国大陆现行法体系与请求权基础方法论。通过互动式推演，它把每一个杂乱无章的案件纠纷，强行压进一条严密的实务主线：诉讼请求怎么选？依据怎么找？债法关系怎么拆？对方怎么打？证据够不够？法官会问什么？  

本项目旨在帮助法律小白、初级律师及法学生，在起草诉状或开庭前，完成一次次高压下的“模拟法庭”兵棋推演。  

💎 核心精华 (The Essence)  
一个底层逻辑：所有的推演都从《民法典》的“给付”切入（谁得向谁，基于何种法律依据，主张何种法律救济）。  
拒绝一次性灌输：采用“教练式一问一答”，一次只追问一个核心痛点，用户卡住时提供递进式帮扶。  
双视角动态攻防：既做原告的“矛”（寻找最经济、负担最轻的请求权基础），也做被告的“盾”（模拟最致命的抗辩事由与履行障碍）。  

🗺️ 七步推演法 (Key Steps)  
本 Agent 的运行轨迹遵循一套严密的“七步检视法”：  
案件压缩 (Case Compression)：剥离情绪废话，锁定核心当事人、可证明事实与最终诉讼目标。  
债法定位 (Debt Mapping)：拆解意定/法定之债，精准识别主给付、从给付与附随义务，规避债权相对性风险。  
诉请筛选 (Claim Selection)：列出主位与备位诉讼请求候选，并进行竞合审查（如违约与侵权竞合）。  
请求权匹配 (Basis Matching)：为首选诉请寻找最直接的实体法依据，拆解产生要件、消灭要件与行使要件。  
胜算排序 (Strategy Sorting)：根据规范直接性、举证难度和诉讼经济性，为策略排定优先级。  
极限攻防 (Stress Testing)：模拟被告的最强抗辩（如时效届满、不安抗辩）与法官的致命追问，倒逼原告回应。  
证据对齐 (Evidence Alignment)：告别按时间线堆砌证据，强制要求“每一份证据必须对应一个构成要件/抗辩事实”，精准识别证据缺口。
